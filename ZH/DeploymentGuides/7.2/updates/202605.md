#### 配置加密

> 注意：只有 `job.encrypt.password` 这个涉及到持久化的数据，修改后能够兼容读取老数据，但是如果需要彻底更新掉默认的旧密码加密的数据的话，需要手动改配置触发下存量数据迁移。
> 

```bash
jobEncryptPassword=$(openssl rand -base64 12 | tr '+/' '_-' | cut -c1-16)
jobArtifactoryPassword=$(openssl rand -base64 12 | tr '+/' '_-' | cut -c1-16)
jobActuatorUserPassword=$(openssl rand -base64 12 | tr '+/' '_-' | cut -c1-16)
read -r jobSm2PrivateKey jobSm2PublicKey < <(
  kubectl -n blueking run bk-job-keypair --rm -i --image hub.bktencent.com/dev/blueking/bk-job-keypair:0.0.1 -- python sm2_keypair/generate_sm2_keypair.py 2>/dev/null | 
  awk '
    /原始私钥:/ {getline; priv=$1} 
    /原始公钥:/ {getline; pub=$1} 
    END {print priv, pub}
  '
)
read -r jobPrivateKeyBase64 jobPublicKeyBase64 < <(
  kubectl -n blueking run bk-job-keypair --rm -i --quiet --restart=Never --image hub.bktencent.com/dev/blueking/bk-job-keypair:0.0.1 -- python service-rsa-keypair/generate_service_rsa_keys.py 2>/dev/null | \
  jq -r '."job.security.privateKeyBase64" + " " + ."job.security.publicKeyBase64"'
)

cd $INSTALL_DIR/blueking/  # 进入工作目录
touch ./environments/default/bkjob-custom-values.yaml.gotmpl

yq -i "
  .job.encrypt.password = \"$jobEncryptPassword\" |
  .job.encrypt.sm2PublicKey = \"$jobSm2PublicKey\" |
  .job.encrypt.sm2PrivateKey = \"$jobSm2PrivateKey\" |
  .job.security.actuator.user.password = \"$jobActuatorUserPassword\" |
  .job.security.privateKeyBase64 = \"$jobPrivateKeyBase64\" |
  .job.security.publicKeyBase64 = \"$jobPublicKeyBase64\" |
  .artifactory.job.password = \"$jobArtifactoryPassword\"
" ./environments/default/bkjob-custom-values.yaml.gotmpl
```

检查输出，如果校验不通过，请根据空字段重新执行上述生成密钥命令后写入文件。
```bash
yq -e '
  (.job.encrypt.password            | length > 0) and
  (.job.encrypt.sm2PublicKey        | length > 0) and
  (.job.encrypt.sm2PrivateKey       | length > 0) and
  (.job.security.actuator.user.password | length > 0) and
  (.job.security.privateKeyBase64   | length > 0) and
  (.job.security.publicKeyBase64    | length > 0) and
  (.artifactory.job.password        | length > 0)
' "$TARGET_FILE" >/dev/null \
  && echo "INFO: 校验通过" \
  || { echo "ERROR: 存在空字段,请检查 ./environments/default/bkjob-custom-values.yaml.gotmpl" >&2; }
```

更新 bk-job：
``` bash
helmfile -f base-blueking.yaml.gotmpl -l name=bk-job sync
```

等待命令执行完毕，结尾输出如下即为更新成功：
>``` plain
>UPDATED RELEASES:
>NAME    CHART            VERSION
>bk-job  blueking/bk-job  0.7.4-beta.4
>```

#### 配置加密

```bash
cd $INSTALL_DIR/blueking/  # 进入工作目录
touch ./environments/default/bkci/bkci-custom-values.yaml.gotmpl

bkCiStoreAesKey=$(openssl rand -base64 32 | LC_ALL=C tr -dc 'A-Za-z0-9!@#$%^&*()_+{}|:<>?~' | head -c 16)
bkCiTicketCertAesKey=$(openssl rand -base64 32 | LC_ALL=C tr -dc 'A-Za-z0-9!@#$%^&*()_+{}|:<>?~' | head -c 16)
bkCiTicketCredentialAesKey=$(openssl rand -base64 32 | LC_ALL=C tr -dc 'A-Za-z0-9!@#$%^&*()_+{}|:<>?~' | head -c 16)
bkCiEnvironmentBatchInstallAesKey=$(openssl rand -base64 32)
bkCiProcessCallbackAesKey=$(openssl rand -base64 32 | LC_ALL=C tr -dc 'A-Za-z0-9!@#$%^&*()_+{}|:<>?~' | head -c 16)
bkCiRepositoryAesGit=$(openssl rand -base64 32 | LC_ALL=C tr -dc 'A-Za-z0-9!@#$%^&*()_+{}|:<>?~' | head -c 16)
bkCiRepositoryAesGithub=$(openssl rand -base64 32 | LC_ALL=C tr -dc 'A-Za-z0-9!@#$%^&*()_+{}|:<>?~' | head -c 16)

yq -i "
  .config.bkCiStoreAesKey = \"$bkCiStoreAesKey\" |
  .config.bkCiTicketCertAesKey = \"$bkCiTicketCertAesKey\" |
  .config.bkCiTicketCredentialAesKey = \"$bkCiTicketCredentialAesKey\" |
  .config.bkCiEnvironmentBatchInstallAesKey = \"$bkCiEnvironmentBatchInstallAesKey\" |
  .config.bkCiProcessCallbackAesKey = \"$bkCiProcessCallbackAesKey\" |
  .config.bkCiRepositoryAesGit = \"$bkCiRepositoryAesGit\" |
  .config.bkCiRepositoryAesGithub = \"$bkCiRepositoryAesGithub\"
" environments/default/bkci/bkci-custom-values.yaml.gotmpl
```

检查配置
```bash
yq -e '
  (.config.bkCiStoreAesKey                      | length > 0) and
  (.config.bkCiTicketCertAesKey                  | length > 0) and
  (.config.bkCiTicketCredentialAesKey            | length > 0) and
  (.config.bkCiEnvironmentBatchInstallAesKey     | length > 0) and
  (.config.bkCiProcessCallbackAesKey             | length > 0) and
  (.config.bkCiRepositoryAesGit                  | length > 0) and
  (.config.bkCiRepositoryAesGithub               | length > 0)
' environments/default/bkci/bkci-custom-values.yaml.gotmpl >/dev/null \
  && echo "INFO: 7个关键配置校验通过" \
  || { echo "ERROR: 存在空字段，请检查 environments/default/bkci/bkci-custom-values.yaml.gotmpl" >&2; }
```

#### 备份数据库

本次升级涉及密钥修改，建议升级前备份蓝盾服务使用的数据库。

升级 aesKey 并开启密钥刷新前，请先备份以下会被刷新任务改写的表：
- devops_ci_process: T_PROJECT_PIPELINE_CALLBACK
- devops_ci_repository: T_REPOSITORY_GIT_TOKEN, T_REPOSITORY_TGIT_TOKEN, T_REPOSITORY_GITHUB_TOKEN
- devops_ci_store: T_STORE_SENSITIVE_CONF, T_STORE_ENV_VAR
- devops_ci_ticket: T_CERT, T_CERT_ENTERPRISE, T_CERT_TLS, T_CREDENTIAL

如果使用蓝盾部署默认自带的 `bk-ci-mysql` 数据库，可以参考以下全量备份命令：

```bash
BACKUP_DIR="/data/bak"

DATE="$(date '+%Y%m%d%H%M')"
# bk-ci-mysql
kubectl -n blueking exec sts/bk-ci-mysql -c mysql -- mysqldump  -uroot  -pblueking --skip-opt --create-options --default-character-set=utf8mb4 -R -E -q -e --single-transaction --no-autocommit --max-allowed-packet=1G --hex-blob --all-databases > $BACKUP_DIR/mysql/mysqldump_bkci_mysql_$DATE.sql
```
