Q:
这个智能体为什么不按照给定的角色 Prompt 输出，而是开始胡言乱语？还能继续用 deepseek-v3 吗？

A:
原因是 deepseek-v3 在调用时存在问题，SDK 侧当时正在适配，所以导致输出异常，不是角色 Prompt 本身失效，也不是 agent 配置被改动引起的。临时处理方案是先切换到 dsv32 或 hunyuan-turbos，这两个模型当时测试效果都可以。

至于还能不能继续用 deepseek-v3，当时后续反馈里提到接口一度“返回 ok 了”，但该智能体的调用量非常大，正常情况下通过 API 每天大约调用 1.2w 次。这个量级已经明显偏高，也直接引出了后续容量问题：deepseek-v3 / 3.1 / 3.2 以及 qwen235b 这类大模型总体容量有限，实例打满后没办法快速扩容。因此，短期内即使 deepseek-v3 接口恢复，也不建议直接依赖它作为高频主模型，更稳妥的方案是优先使用 hunyuan-turbos，或尝试 qwen3-nothinking、gpt-oss 这类更轻量、扩容更充足的模型。
